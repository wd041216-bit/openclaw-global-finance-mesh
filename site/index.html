Why

Zhouheng is built for teams who need deterministic finance decisions, replay-safe rule rollout, governed legal references, and audit credibility in one operator-ready console.

Business-first UX

Role switch home + guided onboarding + cascade subpages.

Trusted operations

SQLite hash-chain ledger, backup replication, and non-destructive restore drills.

Agent-ready integration

OpenClaw native plugin + shared MCP for Claude/Manus/Cursor/Cline/Cherry Studio.

Try

  1. Install and launch desktop app.
  2. Open first-launch onboarding and bootstrap admin.
  3. Set `OLLAMA_MODE=cloud`, `OLLAMA_MODEL=kimi-k2.5`, `FINANCE_MESH_CLOUD_API_FLAVOR=auto`.
  4. Run runtime probe, then example decision and replay.
npm install
npm test
npm run dev

Install

macOS

Official package outputs: `.pkg`, `.dmg`, `.zip`.

First launch opens `getting-started.html?mode=admin&entry=desktop`.

Windows

Official package outputs: `.exe` (NSIS) + `.zip` fallback.

Tray app includes onboarding, workbench, system, recovery, and Agent Hub entries.

Cloud runtime default

`OLLAMA_MODE=cloud` + `OLLAMA_MODEL=kimi-k2.5` + `FINANCE_MESH_CLOUD_API_FLAVOR=auto`.

Connect Agents

One capability surface, multiple host paths.

  • OpenClaw: Native plugin adapter.
  • Claude: Shared MCP connector.
  • Manus: Shared MCP connector.
  • Cursor: Shared MCP connector.
  • Cline: Shared MCP connector.
  • Cherry Studio: Shared MCP connector.

Pilot

Pilot baseline is single-instance Docker + cloud runtime verification gate. Go live only when runtime verification returns `fully_usable` with `goLiveReady=true`.

Validation

`npm run verify:cloud-provider` and `npm run review:pilot`.

Recovery readiness

Backup target required + restore drill required before external access.

Observability

Use `/api/operations/health`, `/api/metrics`, and structured logs.

Download

Release assets are published to GitHub Releases and verified by `SHA256SUMS`.
