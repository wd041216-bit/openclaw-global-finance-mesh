# v0.4.0 Launch Checklist

Use this checklist before cutting tag `v0.4.0`.

## Repository and docs

- [ ] `package.json` version is `0.4.0`
- [ ] `CHANGELOG.md` has `## 0.4.0`
- [ ] README top section reflects download + host matrix + pilot posture
- [ ] README.zh-CN is aligned with the same default runtime and download path
- [ ] `SECURITY.md` and `SUPPORT.md` are present and linked in issue template config

## Build and test gates

- [ ] `npm test`
- [ ] `npm run verify:server`
- [ ] `npm run doctor:hosts`
- [ ] `npm run smoke:ui`
- [ ] `npm run smoke:restore`
- [ ] `npm run smoke:pages`
- [ ] `npm run smoke:release-artifacts` (metadata mode)

## Release workflow dry run

- [ ] Trigger release workflow via `workflow_dispatch` with `v0.4.0`
- [ ] macOS artifacts generated: `.pkg`, `.dmg`, `.zip`
- [ ] Windows artifacts generated: `.exe`, `.zip`
- [ ] `SHA256SUMS` generated and attached
- [ ] `npm run smoke:release-artifacts -- --strict --artifacts-dir <downloaded-release-assets-dir>` passes

## Pages

- [ ] GitHub Pages workflow deployed successfully from `main`
- [ ] Page sections visible: `Why`, `Try`, `Install`, `Connect Agents`, `Pilot`, `Download`
- [ ] Download links match release artifact names

## Pilot-readiness sanity

- [ ] Desktop onboarding opens `getting-started.html?mode=admin&entry=desktop`
- [ ] Runtime gate shows `goLiveReady` semantics clearly in system/workbench
- [ ] Cloud default remains `OLLAMA_MODE=cloud`, `OLLAMA_MODEL=kimi-k2.5`, `FINANCE_MESH_CLOUD_API_FLAVOR=auto`
