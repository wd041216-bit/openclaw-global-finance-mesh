# Security Policy

## Supported versions

Security fixes are provided for the latest published minor version on `main`.

## Reporting a vulnerability

Please do not open a public issue for security-sensitive reports.

Use one of these channels:

- Open a private security advisory in GitHub (preferred)
- Email: `wd041216.bit+security@gmail.com`

Include:

- affected version or commit
- impact summary
- reproduction steps
- proof-of-concept (if available)
- suggested mitigation (optional)

## Response targets

- first acknowledgement: within 72 hours
- triage result: within 7 days
- patch and disclosure timeline: agreed after triage

## Scope notes

- local-first deployment defaults are in scope
- configuration mistakes and missing hardening guidance are in scope
- third-party provider account entitlement limits are out of scope unless Zhouheng misreports them
