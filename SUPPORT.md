## Issue templates

Use GitHub issue forms for:

- runtime or console bugs
- desktop installation and onboarding issues
- host integration issues (OpenClaw / Claude / Manus / Cursor / Cline / Cherry Studio)
- feature requests

## What to include in a support request

- environment (OS, Node version, browser, deployment mode)
- exact command or page action
- expected vs actual behavior
- sanitized logs / screenshots
- relevant config snippets (without secrets)

## Response expectations

- community support is best-effort
- production incident handling is not guaranteed in this beta phase
